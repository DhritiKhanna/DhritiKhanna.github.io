I was assigned the task of optimization of verifying sequentialized concurrent programs. The work on sequentialization translated the parallel program into a sequential program by nondeterministically assigning timestamps to the read and write memory accesses and they encode the semantics of the memory model as constraints on these timestamps. The set of interleavings this sequential program can simulate is same as that of the parallel program. Even though sequentializing a program improves the performance over model checking a parallel program, all the interleavings are explored and the time is not reduced drastically. Although an abstract program represent an over approximation program and there are more interleavings than the original program, model checking an abstract program is easier because it is less constrained. When we replay the counter example on the original program, it may result into a spurious counter example because it does not violate the assertion.
